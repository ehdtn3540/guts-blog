컴퓨터 구조(Computer Architecture)는 **컴퓨터가 어떻게 구성되어 있고, 각 부품이 어떤 역할을 하며, 서로 어떻게 동작하는지**를 설명하는 분야입니다.
소프트웨어(프로그램)가 **하드웨어 위에서 어떻게 실행되는지 이해하는 기초**라고 보면 됩니다.

---

## 1️⃣ 컴퓨터 구조의 큰 그림

![Image](https://upload.wikimedia.org/wikipedia/commons/0/08/Computer_architecture_block_diagram.png?utm_source=chatgpt.com)

![Image](https://upload.wikimedia.org/wikipedia/commons/e/e5/Von_Neumann_Architecture.svg?utm_source=chatgpt.com)

![Image](https://bob.cs.sonoma.edu/IntroCompOrg-x64/book_figs/subsys.png?utm_source=chatgpt.com)

컴퓨터는 크게 다음 4가지로 구성됩니다.

1. **CPU (중앙처리장치)**
2. **메모리 (주기억장치)**
3. **보조기억장치**
4. **입출력 장치 (I/O)**

이 모든 것들이 **버스(Bus)** 라는 통로로 연결되어 있습니다.

---

## 2️⃣ CPU (Central Processing Unit)

![Image](https://www.tutorialspoint.com/computer_fundamentals/images/computer_architecture.jpg?utm_source=chatgpt.com)

![Image](https://webdocs.cs.ualberta.ca/~amaral/courses/329/webslides/TopicH-DataPath/img34.jpg?utm_source=chatgpt.com)

CPU는 컴퓨터의 **두뇌** 역할을 합니다.

### CPU의 주요 구성 요소

* **ALU (산술논리연산장치)**
  → 덧셈, 뺄셈, 비교 같은 연산 수행
* **Control Unit (제어장치)**
  → 명령어를 해석하고 각 부품에 지시
* **Register (레지스터)**
  → 연산에 필요한 아주 빠른 임시 저장 공간

👉 **레지스터 > 캐시 > 메모리 > 디스크** 순으로 속도가 느려집니다.

## 3️⃣ 메모리 (Memory)

![Image](https://tams.informatik.uni-hamburg.de/applets/hades/webdemos/40-memories/40-ram/ram.gif?utm_source=chatgpt.com)

![Image](https://upload.wikimedia.org/wikipedia/commons/0/0c/ComputerMemoryHierarchy.svg?utm_source=chatgpt.com)

### 메모리의 특징

* 실행 중인 프로그램과 데이터를 저장
* 전원이 꺼지면 데이터가 사라짐 (**휘발성**)

### 메모리 계층 구조

1. 레지스터 (CPU 내부)
2. 캐시 메모리
3. RAM
4. SSD / HDD

➡️ **빠를수록 비싸고 용량이 작다**

---

## 4️⃣ 보조기억장치 (Storage)

![Image](https://media.geeksforgeeks.org/wp-content/uploads/20250317145546163469/Difference-Between-HSS-and-SSD.webp?utm_source=chatgpt.com)

![Image](https://computerguidehub.com/wp-content/uploads/secondary-storage-devices-images-names.webp?utm_source=chatgpt.com)

* SSD, HDD 등
* 전원이 꺼져도 데이터 유지 (**비휘발성**)
* 프로그램과 데이터를 장기 저장

👉 프로그램은 **보조기억장치 → 메모리 → CPU** 순으로 이동하며 실행됩니다.

---

## 5️⃣ 입출력 장치 (I/O)

![Image](https://www.vedantu.com/seo/content-images/9658af56-19b0-44f8-ac3f-322499e6b321_1..png?utm_source=chatgpt.com)

![Image](https://media.geeksforgeeks.org/wp-content/uploads/20250315122640215754/Input----Output--Devices.webp?utm_source=chatgpt.com)

* 입력: 키보드, 마우스, 마이크
* 출력: 모니터, 스피커, 프린터

입출력 장치는 CPU보다 느리기 때문에
👉 **인터럽트(Interrupt)** 방식으로 처리합니다.

## 6️⃣ 명령어 실행 흐름 (Fetch → Decode → Execute)

![Image](https://upload.wikimedia.org/wikipedia/commons/d/de/Fetch-Decode-Execute_Cycle.png?utm_source=chatgpt.com)

![Image](https://i.ytimg.com/vi/xs5oq-i_rTc/maxresdefault.jpg?utm_source=chatgpt.com)

CPU는 다음 과정을 반복합니다.

1. **Fetch** – 메모리에서 명령어 가져오기
2. **Decode** – 명령어 해석
3. **Execute** – 연산 수행
4. **Store** – 결과 저장

이 사이클이 초당 수십억 번 실행됩니다.

---

## 7️⃣ 폰 노이만 구조 (Von Neumann Architecture)

![Image](https://upload.wikimedia.org/wikipedia/commons/e/e5/Von_Neumann_Architecture.svg?utm_source=chatgpt.com)

![Image](https://computerscience.gcse.guru/wp-content/uploads/2016/04/Von-Neumann-Architecture-Diagram.jpg?utm_source=chatgpt.com)

* 프로그램과 데이터가 **같은 메모리**에 저장
* 구조가 단순하지만
  👉 CPU ↔ 메모리 병목 현상 발생 (**폰 노이만 병목**)

---

## 8️⃣ 컴퓨터 구조를 왜 배워야 할까?

### 개발자에게 중요한 이유

* 코드가 **왜 느린지** 이해 가능
* 메모리 사용 최적화 가능
* 멀티스레드, 병렬 처리 이해
* OS, 컴파일러, 네트워크 학습의 기반

👉 **“코드는 하드웨어 위에서 실행된다”** 를 이해하게 됨

---

## 9️⃣ 한 줄 요약

> 컴퓨터 구조란
> **CPU가 명령어를 처리하고, 메모리가 데이터를 제공하며,
> 전체 시스템이 어떻게 협력해서 프로그램을 실행하는지를 설명하는 학문**이다.
