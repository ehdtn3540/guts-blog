## 변수 선언 키워드
메모리에 값을 지정하기 위해 사용한다.
- const(Constant):  
재할당이 불가능한 상수를 선언하며 선언과 동시에 초기화해야 한다.  
값이 변하지 않음을 보장하는 가장 권장되는 선언 방식이다.  
변수의 생명주기(Scope)는 함수가 호출될때 독립적인 메모리 공간이 생기며,  
그 공간이 유지되는 동안 딱 한 번만 값을 가질수 있다.

- let:  
재할당이 가능한 변수를 선언하며 값이 바뀔 가능성이 있는 경우에 사용한다.

- var:  
과거에 쓰던 방식이며 호이스팅(Hoisting)등 예상치 못한 버그를 유발할 수 있어  
현대 개발(Next.js 등)에서는 거의 사용하지 않는다.


## 데이터 타입(자료형)
### 데이터 타입 지정의 차이
JavaScript에서는 타입을 지정할 수 없고, TypeScript에서는 타입을 지정하지 않아도  
자동으로 타입을 설정한다. 경우에 따라 TypeScript는 타입을 지정해 줄 수도 있다.


### 원시 타입(Primitive Types): 값이 하나인 단순한 형태
- string: 텍스트
- number: 정수와 실수
- boolean: 참/거짓
- undefined: 변수를 선언만 하고 값을 넣지 않았을 때의 기본값
- null: "값이 없음"을 의도적으로 명시할 때 쓰는 값 
- bigint: 아주 큰 숫자
- symbole: 고유하고 변경 불가능한 식별자

### 참조 타입(Reference Types)
- object: 여러 값을 묶은 것(객체, 배열([]), 함수(function) 모두 여기에 속함)


## TypeScript의 데이터 타입(TS전용)
TypeScript는 JS의 타입들 위에 개발자가 코드를 짤 때 도움을 주는 타입들을 추가했습니다.

### 특수 타입
- any: "아무거나 다 됨", 타입 검사를 포기하는 것이라 꼭 필요할 때만 사용
- unknown: "뭔지 모르지만 일단 저장", 사용하려면 나중에 타입을 확인하는 과정이 필요
- void: 아무것도 반환하지 않는 함수의 반환 타입
- never: 절대 발생할 수 없는 값(예: 무한 루프나 항상 에러를 던지는 함수)

### 복합/구조적 타입
- Array: 특정 타입만 담는 배열(에: number[])
- Tuple: 개수와 순서가 정해진 배열
- Enum: 특정 숫자나 문자열 세트에 이름을 붙인 것
- Union(|): 이것 혹은 저것
- Literal: 특정 값 그 자체(예: "UP" | "DOWN")
