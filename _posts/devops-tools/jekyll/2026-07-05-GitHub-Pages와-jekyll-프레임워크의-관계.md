GitHub Pages와 Jekyll을 활용한 정적 블로그 운영 및 상호 관계에 대해 알아본다.

---

## 1. GitHub Pages란 무엇인가
GitHub Pages는 GitHub에서 제공하는 무료 웹 호스팅 서비스이다.
- 정적 웹사이트 호스팅: HTML, CSS, JavaScript로 이루어진 정적 페이지를 무료로 띄워준다.
    - 정적 페이지 -> 클라이언트 사이드 실행
- 서버 관리 여부: 복잡한 서버 설정이나 비용 관리 없이, 소스 코드만 올리면 웹사이트 생성이 가능하다.
- 도메인 제공: 기본적으로 "본인ID.github.io" 형태의 주소를 무료로 제공한다.


## 2. Jekyll이란 무엇인가
Jekyll(지킬)은 텍스트 파일을 정적 웹사이트로 변환해 주는 정적 사이트 생성기(SSG)이다.
- 마크다운 기반: 복잡한 HTML 코딩 없이, 단순한 글방식인 마크다운(.md) 파일로 글을 쓰면 알아서 예쁜 웹 페이지로 바꾸어 준다.
- 테마 재사용: 레이아웃, 헤더, 푸터 같은 반복되는 구조를 템플릿화하여 관리하기 쉽다.  
- 데이터베이스(DB) 없음: 워드프레스나 티스토리와 달리 데이터베이스가 없다.  
모든 글이 텍스트 파일로 저장되므로 속도가 아주 빠르고 보안이 강력하다.
- Ruby 기반: Jekyll은 Ruby 언어 기반으로 제작된 도구이기 때문에, 설치하거나 테마를 설정할 때 Ruby 환경이 필요하다.


## 3. GitHub Pages와 Jekyll의 상호관계
이 둘은 개별적으로 사용이 가능하지만 GitHub Pages가 Jekyll을 공식적으로 지원하기 때문에 시너지가 발생한다.  
- 자동 빌드 시스템  
마크다운(.md)으로 쓴 글을 GitHub 저장소에 올리면 GitHub Pages가 Jekyll을 공식적으로 지원하기 때문에 자동으로 Jekyll을 빌드하여 웹사이트를 구현해 준다. 
하지만 글 작성시 편집 내용을 바로바로 확인하기 위해선 Jekyll 로컬 환경 구축이 사실상 필수이다.
- 버전 관리와 블로그의 만남  
블로그의 디자인 변경 사항이나 내가 쓴 글의 수정 역사가 GitHub에 고스란히 기록되기 때문에 글을 쓰다가 실수했다면 언제든 과거 버전으로 되돌릴 수 있다.  
- 완벽한 비용 제로(0원)의 정적 환경  
Jekyll이 만들어내는 결과물은 데이터베이스가 필요 없는 순수한 '정적 파일'이고 
GitHub Pages는 '정적 파일 호스팅'에 특화되어 있다. 즉, Jekyll의 결과물과 GitHub Pages의 서비스 성격이 100% 일치하므로 트래픽이 몰려도 막힘없이 빠른 블로그를 무료로 운영할 수 있다.
